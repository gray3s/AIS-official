within a specified radius of St. George, Utah.

were complete.

- Coverage failure
- False completeness

Case #003 - Logical Project Structure Treated as Repository Contents

Context:
A remembered project hierarchy was treated as though it represented actual,
verified repository contents.

Failure:
A logical structure was discussed as if it contained inspectable repository
artifacts.

Consequence:
The user could reasonably infer that repository contents were known when only
a logical project structure was known.

Classification:
- Context conflation
- False repository-state inference


Case #004 - Artifact Recovery State Inconsistency/Inaccuracy

Context:
One conversation reported recovery of the original 14-prompt AIST artifact,
while another conversation could not locate the artifact by filename,
description, or project lineage.

Failure:
Conflicting claims regarding the existence and recoverability of a project
artifact.

Classification:
- Cross-context state inconsistency/inaccuracy
- Artifact provenance failure
- Recovery-verification failure


Observation:

A recurring pattern across these cases is:

Partial information
        ↓
Assumption
        ↓
Presented as complete knowledge

The specific domains differ, but the failure mode is similar.
